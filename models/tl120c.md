# TL120C / TL120C-2

A 120 cm RGB tube in Neewer's Infinity tube line. The TL120C (light-type 50) and TL120C-2
(light-type 101) are equivalent in behaviour; the `-2` is a later firmware generation.

| | |
|---|---|
| Light-type | 50 (TL120C), 101 (TL120C-2) |
| Advertised serial | `NW-20230031` (TL120C), `NW-20240047` (TL120C-2) |
| Latest firmware | 1.2.8 (TL120C), 2.0.5 (TL120C-2) |
| CCT range | 2500–10000 K |
| Battery pack | 10.8 V / 6000 mAh (confirmed) |
| Charger | 20 V / 5.0 A / 100 W barrel PSU, Neewer model **NT136-2000500D2** ([label](../assets/neewer-psu-nt136-2000500d2-20v-5a-100w.jpg)) — higher-amperage than the 90C brick, same 20 V |
| Wired DMX512 port | Yes (via NC005/NC006 RJ45-to-XLR cable) |
| Wireless DMX (2.4G) | No |

## Capabilities

- **HSI, CCT (with GM), RGBCW, CIE xy, gel** — full colour.
- **Pixel effects** (`pixelEffectClassify` 1) — the palette-band effect system. See
  [../pixel-and-streamer.md](../pixel-and-streamer.md).
- **No Streamer / FX-Flow** — that feature is TL60-only.

## Protocol notes

- **Extended colour is by-MAC only.** The direct forms of RGBCW (`0xA8`), xy (`0xB9`), gel
  (`0xAF`), and scene (`0x88`) are silently ignored. Use the by-MAC opcodes: RGBCW `0xA9`, xy
  `0xB7`, gel `0xAD`, scene `0x91`. Basic power (`0x81`), HSI (`0x86`), and CCT (`0x87`)
  work as direct frames.
- **Built-in scenes use `0x91`, not `0x88`.** The TL120C ignores the legacy `0x88` scene
  opcode. Run scenes by-MAC through `0x91` (inner sub-op `0x8b`); the effect parameters match
  the `0x88` catalogue byte-for-byte.
- **Colour-accept ACK (`0x05`).** After it accepts and applies a colour frame, the TL120C-2
  emits a `0x05` reply pair; it stays silent when it ignores a frame. This is a headless
  accept/ignore signal, and it is specific to the TL120C-2 (the TL90C does not do it).
- **Does not answer several queries.** The direct version read (`0x80`), the temperature/fan
  query (`0xB3`), and the streamer-support query (`0xC4`) get no reply on the TL120C. Use the
  by-MAC version read (`0x9E`) instead.
- **On mains power**, the battery reply (`0x05`) reports `0xF0` as a power-source flag rather
  than a 0–100 percentage.

## Pixel effects

The TL120C is pixel-capable via `0xB0` (by-MAC). An effect is an effect id plus scalar
parameters plus a palette of up to eight HSI/CCT colour triplets, rendered as colour bands
along the tube. It is spatial but not a per-LED framebuffer — you pick an effect and a
palette, not raw pixel bytes. Full detail and example frames are in
[../pixel-and-streamer.md](../pixel-and-streamer.md).

## Firmware

Firmware updates use the custom `0x78` block OTA protocol (128-byte blocks). See
